Overview  

This page describes the work done to reorganize Wicket Stuff, and get the projects that are currently building against Wicket 1.4 into a common build and release cycle that will be kept in sync with Wicket.  The goal is to have numbered releases that match Wicket, so that if you depend on Wicket 1.4.1,  you can also depend on "some-wicketstuff-project" 1.4.1.  Certain conventions were also created, such as project / examples being grouped under a common project-parent module, as well as removing "wicket-", "wicketstuff-", and "wicket-contrib-" from all of the project names.
