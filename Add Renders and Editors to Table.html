Wicket Stuff Wiki : Add Renders and Editors to Table
This page last changed on Sep 02, 2010 by pedrosans.

It is possible to customize the way that the cells will to be rendered. The TableModel method getColumnClass is queried to determine what render will to be used to display an table value.
By default table uses an TextField or Label to render numbers and strings, and a CheckBox to booleans.

That can to be customized like:

table.setDefaultRenderer(MyClass.class, myRender);
table.setDefaultEditor(MyClass.class, myEditor);

To determine which render/editor will to be used, table has an column model also. If at an column need an custom render, that can be done like:

table.getColumnModel().addColumn(new SelfUpdateColumn(2, Duration.seconds(5)));

in this example SelfUpdateColumn is an TableColumn object added to table ColumnModel. The TableColumn provides an render and an editor that will to be used both to display and edit the values in this column.

Example

An TableModel implemantation can specify that at column 0 an boolean value is displayed.

@Override
public Class<?> getColumnClass(int columnIndex)
{
    if (columnIndex == 0)
    {
        return Boolean.class;
    }
    else
    {
        return super.getColumnClass(columnIndex);
    }
}

It is enough to Table component use an render for boolean values
